# Обновление прошивки погрузчика: имя Wi-Fi в телеметрии

## Что входит в эту сборку

Файл прошивки:

```text
build\esp32.esp32.nodemcu-32s\jun30a.ino.bin
```

SHA-256:

```text
AA4351D54D11C953BABCA59FEEEF33226909F965CC7F84B483490E5A61A07862
```

После обновления поле JSON `wifi_profile` в телеметрии будет содержать
фактически подключённую сеть: `ISRK_Hozyain` либо `Sasung`. При отсутствии
подключения оно равно `disconnected`. Адрес сервера, идентификатор погрузчика,
пароли Wi-Fi и порядок подключения не изменены.

## Перед началом

1. Погрузчик должен быть включён и подключён к сети Wi-Fi Raspberry Pi.
2. Ноутбук должен иметь доступ к Raspberry Pi по Amnezia: `10.8.1.2`.
3. На Raspberry Pi должен быть установлен Python 3.
4. Не выключайте питание ESP32 в процессе OTA-обновления.

## 1. Проверить файл на ноутбуке

В PowerShell из папки этой инструкции выполните:

```powershell
$bin = "C:\Users\Windows\projects\site_korovki\!LOADER_CODE\jun30a\build\esp32.esp32.nodemcu-32s\jun30a.ino.bin"
Get-FileHash $bin -Algorithm SHA256
```

Значение `Hash` должно совпасть с указанным выше.

## 2. Передать файл и утилиту OTA на Raspberry Pi

В PowerShell:

```powershell
$bin = "C:\Users\Windows\projects\site_korovki\!LOADER_CODE\jun30a\build\esp32.esp32.nodemcu-32s\jun30a.ino.bin"
$espota = Get-ChildItem "$env:LOCALAPPDATA\Arduino15\packages" -Recurse -Filter espota.py |
  Select-Object -First 1 -ExpandProperty FullName

$espota
ssh isrk@10.8.1.2 "mkdir -p /home/isrk/esp32-ota"
scp $bin isrk@10.8.1.2:/home/isrk/esp32-ota/jun30a-wifi-ssid.bin
scp $espota isrk@10.8.1.2:/home/isrk/esp32-ota/espota.py
```

Если строка `$espota` пуста, в Arduino IDE установите пакет плат **esp32 by
Espressif Systems** через Board Manager, затем повторите команду.

## 3. Найти погрузчик в сети Raspberry Pi

Подключитесь к Raspberry Pi:

```bash
ssh isrk@10.8.1.2
cd /home/isrk/esp32-ota
ls -lh
ping -c 3 rtk-loader-01.local
```

Если `ping` по имени не проходит, найдите IP устройства:

```bash
ip neigh show dev wlan0
arp -a
avahi-resolve -n rtk-loader-01.local
```

Используйте найденный IP на следующем шаге вместо имени
`rtk-loader-01.local`.

## 4. Загрузить прошивку по OTA

На Raspberry Pi:

```bash
python3 /home/isrk/esp32-ota/espota.py \
  -i rtk-loader-01.local \
  -p 3232 \
  -f /home/isrk/esp32-ota/jun30a-wifi-ssid.bin
```

При загрузке по IP замените только аргумент после `-i`, например:

```bash
python3 /home/isrk/esp32-ota/espota.py \
  -i 10.42.0.166 \
  -p 3232 \
  -f /home/isrk/esp32-ota/jun30a-wifi-ssid.bin
```

Успешное завершение `espota.py` означает, что ESP32 получила прошивку и
перезагрузится. Не ориентируйтесь на `nc -vz ... 3232`: для ArduinoOTA он
может показывать `Connection refused` даже при исправной OTA-загрузке.

## 5. Проверить после перезагрузки

Подождите около 20 секунд, затем на Raspberry Pi выполните:

```bash
ping -c 3 rtk-loader-01.local
```

В журнале телеметрии сервера у новых пакетов от `rtk_loader_01` проверьте
значение поля `wifi_profile`. Ожидаемое значение — имя сети, например:

```json
"wifi_profile":"ISRK_Hozyain"
```

или:

```json
"wifi_profile":"Sasung"
```

Если есть Serial Monitor, после перезагрузки ожидается строка вида:

```text
WiFi connected: ISRK_Hozyain (primary)
OTA ready: rtk-loader-01 at 192.168.x.x
```

Метка `primary` в строке Serial Monitor остаётся только внутренним служебным
названием профиля. В отправляемой телеметрии вместо неё будет SSID.
